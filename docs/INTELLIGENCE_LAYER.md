# Intelligence Layer — Name Card Designer

## Messy Inputs
- Free-text company name (may include punctuation, abbreviations, Ltd/Pte)
- Optional address (inconsistent formats)
- Optional back-description (any length, any style)

## Auto-Structure on Submission
```json
{
  "full_name": "Sarah Tan",
  "initials": "ST",
  "company_short": "Tan & Associates",
  "phone_formatted": "+65 9123 4567",
  "has_company": true,
  "has_address": true,
  "has_back_text": true,
  "industry_guess": "legal",
  "industry_confidence": 0.82,
  "industry_source": "keyword-match",
  "industry_review_status": "unreviewed"
}
```

## Events to Track
- Form submitted
- Designs rendered (count rendered, time taken)
- Designs selected (which template_keys chosen)
- Checkout started
- Payment completed / failed
- Confirmation email sent / failed

## Scoring Rules (v1 — rule-based)
- Designs ranked by industry match: if `industry_guess=legal` → classic/formal templates ranked first
- If no company name → personal/minimal templates ranked first
- Confidence threshold: scores below 0.5 → neutral sort order, flag for review

## v1 vs Later
- **v1:** rule-based industry detection (keyword match on company name), deterministic template ranking
- **Later:** GPT-4o reads back_description + company name to suggest layout tone; style_summary generated by LLM; user preference learning from past purchases
